Git + GitHub Workflow

Push. Share.
Fork. Collaborate.

A step-by-step guide to publishing a static website on GitHub and forking repositories like a pro.

01
Initialize your local repository Set up Git inside your project folder

Navigate to your project folder and initialise a new Git repository. This creates a hidden .git folder that tracks all your changes.

terminal
$cd my-static-site
$git init
ⓘ Tip: You only need to run git init once per project. After this, Git watches every file change inside the folder.
02
Stage and commit your files Tell Git which files to track

Add your files to the staging area and create your first commit — a snapshot of your project at this point in time.

terminal
# Stage all files
$git add .
# Create your first commit
$git commit -m "Initial commit: add static website files"
ⓘ Tip: Use git status before staging to see which files have changed. Use git add filename.html to stage individual files.
03
Create a repository on GitHub Set up the remote destination

Go to github.com, click the + icon → New repository. Give it a name, choose Public, and do NOT initialise with a README (since you already have local files).

terminal — connect local to remote
# Link your local repo to GitHub
$git remote add origin https://github.com/your-username/your-repo-name.git
# Rename default branch to main
$git branch -M main
04
Push to GitHub Upload your files to the remote

Push your committed files up to GitHub. The -u flag sets the upstream so future pushes only need git push.

terminal
$git push -u origin main
ⓘ Future updates: After the first push, just use git add . && git commit -m "message" && git push to sync new changes.
05
Enable GitHub Pages Make your site live on the web

In your GitHub repository, go to Settings → Pages. Under "Branch", select main and click Save. Your site will be live at:

your live url
https://your-username.github.io/your-repo-name/
ⓘ Note: It may take 1–2 minutes for GitHub Pages to build and publish your site after enabling it.
🔗

Forking a Repository

Copy someone else's repo to your own account and customise it

1
Go to the repository on GitHub — Navigate to the repo you want to fork, e.g. github.com/username/repo
2
Click the Fork button — Top-right corner of the repo page. Select your account as the destination. GitHub creates a copy under your account.
3
Clone your fork locally — Now pull it down to your machine to start editing.
terminal
$git clone https://github.com/your-username/forked-repo.git
$cd forked-repo
4
Add the original repo as upstream — This lets you pull in future updates from the source.
terminal
# Link to original (upstream) repo
$git remote add upstream https://github.com/original-owner/repo.git
# Sync updates from original any time
$git fetch upstream
$git merge upstream/main
5
Push your changes to your fork — Edit, commit, and push as normal. Your changes go to your fork, not the original.
terminal
$git add .
$git commit -m "My changes to the fork"
$git push origin main

Quick reference — common commands

git status
Shows which files are staged, modified, or untracked
git log --oneline
Lists all commits in a compact one-line format
git diff
Shows exactly what changed line-by-line before staging
git pull
Fetches and merges remote changes into your local branch
git branch -a
Lists all local and remote branches in the repo
git clone <url>
Downloads a full copy of a remote repository locally
